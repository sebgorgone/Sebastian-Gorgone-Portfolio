"Turning Cubes"

Turning cubes was the Final Project for my Intro to Visual Programming for the school of Digitial Media and Design at the University of Connecticut.

In this class we had been lectured on the fundementals of javascript using the p5.js library. I learned a handful of javascript concepts in this concepts, but when it came time for the final I was on my own. This project had been inspired by a 3Blue1Brown Youtube video, where he shows a grid of rhombi arranged to look like 3d blocks. He does this to solve a 2d puzzle in 3d, however I was intrigued by the geometry described to construct the rhombus grid. I chose to attempt to recreate that grid with a generative and random pattern for my final project.

3Blue1Brown Video

Initial Iteration

My first go through I painstakingsly put together an itertive grid of rhombus objects. Each rhombus was logically bound to a given 'center point' drawn at each one of the grid points. The result was a clean meshing grid of rhombi that looked like cubes, however this approach got me in trouble when it came to generatively growing the pattern of the cube structure.

This gave the effect of cubes dissappering and reappearing, but the patterns would no go passed a zigzag stairstep pattern. I wanted it to be able to morph into bigger structures for a lack of a better way of putting it.

The Directionality Problem

when building the grid, I built each rhombus point out from one of the obtuse angles using a common radius length. I used the polar cordinates of the points relative to its one anchor point to convert into cartesean cordinates for rendering

As previously mentioned this caused me weeks of roadblock, but I did not remove this as a core principle of how my program works. I tore down all of the code to check for common centers forming a "cube" and wrote functions to append every valid center regardless of polarity to an array. Then using that array as its references it loops over the p5Draw function finding every center and aligning a random center to its three rhombi polarities.

Only then does it perform any tranformations allowing for the generative behavior I had intended.

The Debug display shows you the directionality of the blocks as well as how the state of the canvas changes in regards to the alignment of the rhombi.

It Displays all of the valid centers proving how volitile the lifespan of a given center is.


As a proof of concept outside of my final project. I had wanted to make an initial grid that was one big cube. I noticed when watching my program progress to higher order structures, that bigger and bigger cubes would form. So I decided to do just that, and it is satisfying seeing it work in the other direction.
